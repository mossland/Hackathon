# SmartAI Fridge – AI 식단·식재료 관리 및 자동 장보기 연동 냉장고 시스템

---

## 아이디어 제목
SmartAI Fridge – 바코드 인식 + 키보드·음성 입력 + 마켓 연동 기반 AI 식품관리 냉장고

---

## 간략한 설명
SmartAI Fridge는 바코드 스캐너, 키보드, 음성 인식 기능이 탑재된 스마트 냉장고 모듈로, 냉장고 앞면에 바코드 인식기와 입력 키보드를 탑재하여 식품 정보를 자동 등록하고 유통기한을 관리하며, 보관 중인 재료 기반으로 AI가 레시피와 맞춤형 식단을 추천하는 시스템입니다. 바코드가 없는 식품은 키보드 및 음성 인식으로 수동 입력 가능하며, 유통기한 임박 시 자동 알림이 제공됩니다.여기에 마트·온라인몰 API 연동 기능을 추가해, 식재료가 부족해지면 자동으로 장보기 리스트를 생성하고 직접 구매까지 가능하게 하여 냉장고 자체가 AI 주방 매니저 역할을 수행합니다.

---

## 해결하고자 하는 문제
- 냉장고 속 식재료를 유통기한을 일일이 기억하거나 수동으로 관리하는 번거로움
- 유통기한 초과로 인한 음식물 낭비
- 보유 식재료로 어떤 음식을 만들지 몰라 고민하는 사용자 문제
- 바쁜 일상 속에서 효율적으로 냉장고를 활용하지 못하는 가정이 많음
- 건강 상태에 맞는 식단을 구성하고 재료를 일일이 구매하는 번거로움
- 가족 구성원 간 식재료 정보 공유의 어려움 / 장보기 누락, 중복 구매 등의 문제
- 지속적인 식품 소비 패턴 분석 및 재구매 추천 부재
- 식단 기반 식재료 주문 자동화가 되지 않아 요리 실행률 저하

---

## 제안하는 솔루션

### 1. 스마트 입력 시스템
- 사용자가 냉장고에 식품을 넣을 때 앞문의 바코드 인식기에 스캔하면 자동으로 제품명, 유통기한, 구매일 등이 데이터베이스에 등록됨
- 바코드 데이터는 공공 식품DB 또는 자체 식품DB와 연동하여 자동 매칭 처리
- 바코드가 없는 재료(야채, 생선 등)는 키보드 또는 음성 입력으로 수동 등록
- 사용자의 입력 패턴 학습 후 AI가 자주 입력되는 식품에 대해 자동 제안 기능 제공

---

- 바코드 기반 식품 등록
  - 입력: 사용자가 식품 봉지의 바코드를 스캔
  - 처리: 제품명, 브랜드, 용량, 유통기한 등 자동 조회 / 클라우드 식품 정보 API (예: Open Food Facts, GS1) 연동
- 키보드 + 음성 인식 수동 입력
  - 음성 처리 엔진: Google Speech-to-Text API, Whisper (open-source) 중 선택
  - 예시 명령어: “사과, 유통기한 3일”, “닭가슴살 2팩, 2025년 5월 3일까지”
  - 자연어 파싱: GPT API 또는 미니 NLU 모델로 날짜, 품목명, 수량 추출
  - 자동 추천: 자주 등록한 품목 자동 완성

---

### 2. AI 식재료 모니터링
- 대시보드 화면에 유통기한 임박 식품 자동 정렬 표시
- 스마트폰 연동 앱으로 푸시 알림 전송 가능 (1일/3일/7일 전 설정 가능)
- 경과된 식품은 "폐기 권장" 표시 및 소리/색상으로 구분
- 소비 후 자동 차감 또는 삭제, 남은 재고 기반 리포트 생성
- 스마트폰 앱과 연동하여 집 밖에서도 냉장고 상태와 레시피 확인 가능
- 임박 품목 실시간 하이라이팅 및 알림 (냉장고 디스플레이 + 모바일 앱 연동)
---

- 유통기한 모니터링:
  - 유통기한 D-3, D-1, D-DAY 알림 표시
  - 화면 색상 변화 + 아이콘 표시 (빨강: 임박, 노랑: 주의)
- 중복 재료 병합:
  - 동일 품목이 여러 개 등록되면 수량 기반 병합 및 유통기한 순 정렬
- 유통기한 알림 시스템
  - 냉장고 디스플레이 : 실시간 경고 배너 + 아이콘
  - 모바일 앱:
    - 푸시 알림: "계란 유통기한 내일입니다. 오늘 사용하세요!"
    - 앱 내 위젯 형태 알림 제공 (Android/iOS)
 
 ---

### 3. AI 맞춤형 레시피 & 식단 추천
- 보유 식재료 기반 레시피 자동 추천
- 유통기한이 임박한 재료 위주로 레시피 우선 추천하여 낭비 최소화
- 사용자의 건강 정보에 따른 개인화 식단 제공
- 예시: "저탄고지", "다이어트 중", "채식 위주", "고혈압 환자" 등 태그 기반 맞춤형 식단 필터링
- 주간 맞춤 식단표 자동 생성
- 식단/영양 기준은 WHO 권고 기준 및 외부 건강 DB(API)와 연동 가능
- 사용자는 레시피별 "요리했어요" 버튼 클릭 시 자동으로 재료 소진 처리됨
- 다 쓴 재료는 “소진” 상태로 표시되며 삭제 또는 재구매 제안 가능
- 추천 식단에 필요한 재료와 현재 냉장고 내 재료를 비교하여 "부족한 항목" 자동 탐지

---

- 엔진 구조:
  - OpenAI API 또는 자체 fine-tuned GPT 모델 활용
  - 냉장고 재료 목록을 프롬프트로 사용하여 레시피 자동 생성
- 필터링 옵션:
  - 남은 시간 기준 (유통기한 임박 순)
  - 칼로리/시간/음식 종류(한식/양식 등) 기반 필터
- 디스플레이:
  - 메인 화면: 현재 보관 중인 재료 + 임박 재료 + 오늘의 추천 요리
  - 하단 메뉴: 스캔 / 입력 / 레시피 / 리포트 / 설정
- 모바일 앱 연동:
  - 동일 정보 동기화
  - 집 밖에서도 재료 확인 및 알림 수신
  - 원격으로 가족과 재료 정보 공유

---

### 4. 마트/온라인몰 연동 장보기 기능
- 부족하거나 소진된 재료를 자동으로 장보기 리스트에 추가
- 마켓 API 연동(예: 마켓컬리, 쿠팡, 이마트몰 등) → 가격 비교 및 구매 가능
- “즉시 구매” 또는 “장바구니 담기” 기능 제공
- 정기구매 추천, 반복 품목 자동 예측 및 알림

### 5. 리포트 및 자동화 기능
- 주간/월간 소비 리포트를 통해 식품 소비 패턴 분석
- 자주 소비되는 품목은 자동 재구매 추천 (API 연동 쇼핑몰 또는 스마트 리스팅 기능)
- 자주 쓰는 식품 자동 정기구매 제안

|-----------|-----------|
| 바코드 DB 불완전성 | 공공DB + 자체 사용자 입력 기반으로 지속 보완, 인기 품목 자동 제안 기능 탑재 |
| 사용자 입력 번거로움 | 스마트폰 앱 연동으로 입력 부담 완화 |

| AI 정확도 및 계산 비용 | 클라우드/로컬 연산 분리 및 경량화 모델 사용 |
| UI 진입 장벽 | 직관적인 아이콘 및 레이아웃 구성, 사용자 튜토리얼 제공 |
| 쇼핑몰 API 연동의 복잡성 | 초기 1~2개 마켓 우선 연동, 이후 파트너십 확장 |
| 실시간 재고 정확도 확보 | 사용자 피드백 기반 재고 정정, 센서 연계 기능 확장 고려 |
| 개인정보·구매정보 처리 | 데이터 익명화, 사용자 동의 기반 API 연동 구조 설계 |
| AI 추천의 신뢰성 확보 | 반복 학습을 통한 개인 맞춤 정확도 개선, 사용자 평가 시스템 도입 |
| MOC에 익숙하지 않은 사용자 | 법정화폐 또는 스테이블코인과  가능 UX 설계 (MOC 자동 충전 기능 포함) |
